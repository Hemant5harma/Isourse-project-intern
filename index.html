CI/CD Pipeline using Jenkins and Google Kubernetes Engine (GKE)

🔧 Project Overview

This project implements a complete DevOps CI/CD pipeline using Jenkins for automation and Google Kubernetes Engine (GKE) for cloud-native deployment.

🚀 Tools & Technologies

  • Jenkins: CI/CD automation server
  • Docker: Containerize application
  • GCP: Cloud provider for hosting Jenkins & deploying app
  • GKE: Kubernetes cluster to run containers
  • kubectl: CLI to manage Kubernetes resources
  • GitHub: Source code hosting and integration

⚙️ CI/CD Pipeline Flow

  1. Developer pushes code to GitHub
  2. Jenkins detects the push and triggers the pipeline
  3. Builds Docker image and pushes to Google Artifact Registry
  4. Authenticates with GCP and updates Kubernetes deployment on GKE
  5. Verifies successful rollout

📂 Jenkins Pipeline (Jenkinsfile)

Example pipeline stages:


pipeline {
  agent any
  stages {
    stage('Checkout') { steps { git 'repo-url' } }
    stage('Build') { steps { sh 'docker build -t gcr.io/project/image:$BUILD_NUMBER .' } }
    stage('Push') { steps { sh 'docker push gcr.io/project/image:$BUILD_NUMBER' } }
    stage('Deploy') {
      steps {
        sh 'gcloud container clusters get-credentials...'
        sh 'kubectl set image deployment/app app=gcr.io/project/image:$BUILD_NUMBER'
      }
    }
  }
}

💡 Error Handling Example

When encountering a 502 Bad Gateway error:

  1. Check if the app container is healthy using kubectl logs
  2. Inspect Load Balancer or Ingress configurations
  3. Verify environment variables and service definitions

📈 Benefits

  • Full automation of build, test, deploy
  • Scalable infrastructure on Kubernetes
  • Quick rollback and monitoring support
